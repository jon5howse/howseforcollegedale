# PB-003

# Navigation & Information Architecture

**Project:** Project Beacon

**Epic:** Public Website

**Status:** Draft

**Version:** 1.0

---

# Purpose

The navigation system should help residents quickly find the information they need while naturally encouraging exploration of related topics.

Navigation should feel intuitive, predictable, and uncluttered.

The goal is to organize information around the questions residents ask—not around the internal structure of city government.

---

# Navigation Philosophy

Navigation should answer questions.

Not display categories.

Every menu item should represent a resident's mental model.

# Navigation Goals

Visitors should never need more than three clicks to find any major piece of information.

Navigation should remain consistent throughout the entire website.

The navigation bar remains fixed while scrolling.

---

# Mobile Navigation

On phones, use a hamburger menu.

Navigation should open as a full-height slide-out panel.

Large touch targets.

Clear typography.

No nested menus deeper than one level.

# Official City Resources

Project Beacon supplements—not replaces—the City of Collegedale website.

Whenever residents need official services, Project Beacon should direct them to the City's website.

Examples

Permits

Departments

Utilities

Taxes

Employment

City Services

Official Forms

# Success

Residents never feel lost.

Navigation should disappear into the background.

The focus should remain on understanding the community—not figuring out how to use the website.

Every navigation decision should answer one question:

**Does this help residents find information more naturally?**
