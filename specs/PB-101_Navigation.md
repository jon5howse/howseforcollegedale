# PB-101 — Build Global Navigation

## Status
Ready for Development

## Depends On
- PB-003 Navigation & Information Architecture
- 02_Digital_Experience_Guide.md

## Objective
Build the global navigation system for Project Beacon.

The navigation should be clean, mobile-first, accessible, and consistent across every page.

## Primary Navigation Links
- Home
- Updates
- Projects
- Community CARE
- Laura's Work
- Gallery
- Contact

## Required Components
- Logo / brand mark
- Desktop navigation
- Mobile hamburger menu
- Sticky header
- Community CARE announcement strip
- “Stay Connected” call-to-action button

## Design Requirements
- Navy primary background or white background depending on page context
- Gold accent for active states
- Red reserved for primary action button
- Large touch targets on mobile
- Smooth open/close mobile menu animation
- No dropdown menus for now

## Mobile Requirements
- Hamburger menu
- Full-screen or slide-out menu
- Large readable links
- Easy one-thumb interaction
- Clear close button

## Accessibility Requirements
- Keyboard navigable
- ARIA labels for mobile menu
- High contrast
- Focus states visible
- Screen-reader friendly

## Definition of Done
- Navigation appears on every page
- Mobile menu works
- Links are centralized in one data file
- Styling matches Project Beacon standards
- No broken layout on phone, tablet, or desktop
