# Project Beacon

## Digital Experience Guide

**Version:** 1.0 (Draft)

---

# Purpose

This document defines the visual language, interaction standards, and user experience principles that guide the design of Project Beacon.

Every page, component, and feature should follow these standards.

The goal is consistency.

---

# Design Philosophy

Project Beacon should feel like a modern civic publication rather than a traditional government website.

The experience should communicate:

* Trust
* Community
* Transparency
* Professionalism
* Warmth

Visitors should enjoy exploring the site.

---

# Design Language

## Collegedale Modern

Project Beacon uses a design language called **Collegedale Modern**.

Inspired by:

* Apple
* National Geographic
* Stripe
* Modern magazine design

Characteristics include:

* Large photography
* Clean typography
* Generous white space
* Minimal distractions
* Thoughtful animations
* Calm color palette
* Storytelling through imagery

---

# Mobile First

Project Beacon is designed for phones first.

Expected visitors:

Phone

70–85%

Tablet

10–20%

Desktop

10–20%

Desktop layouts should expand naturally from the mobile experience.

---

# One Thumb Rule

Every important interaction should be reachable with one thumb on a modern smartphone.

Buttons should be large.

Spacing should be generous.

Menus should be simple.

No tiny links.

---

# Responsive Design

Every page must work on:

* iPhone
* Android
* iPad
* Tablets
* Laptop
* Desktop

No feature should exist only on desktop.

---

# Photography

Photography is the primary storytelling element.

Preferred subjects:

* Police
* Fire
* EMS
* Public Works
* Teachers
* Students
* Volunteers
* Businesses
* Veterans
* Parks
* Families
* Community events
* Community CARE meetings
* Laura interacting naturally with residents

Avoid:

Campaign-style portraits

Artificial poses

Stock photography whenever possible

---

# Hero Photography

The homepage hero supports a rotating collection of images.

The collection should represent the many people who contribute to Collegedale.

Rather than highlighting one individual, the hero should celebrate community engagement and public service.

This rotating collection will become known as:

## Faces of Collegedale

Hover

# Accessibility

WCAG AA minimum.

Keyboard navigation.

High color contrast.

Alt text for all images.

Readable typography.

Accessible forms.

# Performance

Largest Contentful Paint under 2.5 seconds.

Images

Fonts

Animations

JavaScript bundles

# Progressive Web App

Project Beacon should eventually support installation on:

iPhone

Android

Desktop

Residents should be able to save the website to their home screen without downloading an app.

---

# Homepage Philosophy

The homepage should answer three questions within five seconds:

1. What is this website?

2. Why should I trust it?

3. Why should I come back?

---

# Faces of Collegedale

The homepage hero should rotate through photographs of people who strengthen the community.

Examples include:

Police

Firefighters

EMS

Teachers

Students

Veterans

Business owners

Volunteers

Public Works

City staff

Families

Laura

The homepage demonstrates leadership by showing Laura serving alongside others rather than standing apart from them.

---

# Community Before Politics

Project Beacon is not a campaign website.

Community stories come first.

Laura's work is demonstrated through participation, transparency, and documented public service.

---

# Design Test

Before publishing any new page, ask:

Does this page help residents better understand their community?

Does it strengthen trust?

Does it encourage engagement?

If not, redesign the page.

---

# Final Principle

Technology should disappear.

Residents should never think about the software.

They should think about their community.
